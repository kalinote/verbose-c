# O1 优化技术说明

本文档说明 Verbose-C 当前 `O1` 优化的功能范围、实现原理和主要算法。`O1` 的设计目标是：在不改变程序可见语义的前提下，利用类型检查后的 AST 信息和已解析的字节码信息，执行一组保守的局部优化。

---

## 1. 总体设计

`O1` 由两层优化组成：

```text
类型检查后的 AST
  -> typed AST 优化
  -> 字节码生成与标签解析
  -> 字节码窥孔优化
```

typed AST 优化负责需要类型、符号表、作用域和副作用判断的优化，例如常量折叠、常量传播、拷贝传播、分支裁剪、公共子表达式消除和简单内联。

字节码优化负责低层控制流清理，例如删除 `NOP`、删除不可达指令、删除无意义跳转和重定向跳转链。该层不再推断类型、变量作用域或表达式副作用。

这种分层避免了在字节码层反推高级语义，也让 AST 层可以充分利用类型检查阶段已经建立的符号表和隐式转换信息。

---

## 2. AST 优化状态

typed AST 优化以过程内、顺序遍历为主。优化器维护一个局部环境 `_OptimizationEnv`：

```text
constants: 变量标识 -> 运行时常量对象
copies:    目标变量标识 -> 源变量标识
```

变量标识优先使用符号对象身份；全局变量等特殊场景可退化为名称标识。

优化环境只表示“当前路径上可以证明成立”的事实。遇到不能证明安全的边界时，优化器会清除部分或全部状态，而不是尝试激进合并。

常见失效边界：

- 变量重新赋值会清除该变量的常量状态和相关拷贝关系。
- 复合赋值、自增、自减会清除被写变量及其依赖关系。
- 指针写入、数组写入、字段写入、对象属性写入会清空可见状态。
- 函数调用默认视为有副作用；取地址实参会触发更强的状态清空。
- 分支和循环不做跨路径固定点分析，无法静态裁剪时不把内部传播结果合并回外层。

---

## 3. 常量折叠

常量折叠将编译期可求值的表达式替换为内部 `ConstantValueNode`。该节点承载已经求值完成的运行时对象，例如 `VBCInteger`、`VBCFloat`、`VBCBool`、`VBCString`、`VBCNull`。

支持的表达式包括：

- 基础字面量：整数、浮点、布尔、字符串、`null`。
- enum 成员常量。
- 一元运算：`+`、`-`、`!`。
- 二元运算：算术运算、比较运算、逻辑运算。

折叠过程不直接使用 Python 原生算术替代语言语义，而是尽量复用现有 VBC 运行时对象的运算行为。这样可以保持整数除法、类型提升、字符串拼接和布尔转换等细节与 VM 一致。

如果求值可能改变运行期行为，优化器会跳过折叠。例如：

- 除零或取模除零。
- 运算类型不支持。
- 表达式包含函数调用。
- 表达式包含取地址、解引用、数组下标、字段访问或对象属性访问。

常量折叠是后续优化的基础。被折叠成 `ConstantValueNode` 的表达式可以继续参与常量传播、分支裁剪和内联后的二次简化。

---

## 4. 常量传播

常量传播记录变量在当前路径上可证明的常量值，并在读取点替换为 `ConstantValueNode`。

典型形式：

```text
int a = 3;
return a + 1;
```

优化环境记录 `a -> 3`，后续读取 `a` 时可替换为常量。替换后表达式可能继续被常量折叠为单个结果。

常量传播采用保守的前向数据流策略：

1. 声明或赋值右值先递归优化。
2. 如果右值可以求为常量，则更新目标变量的常量状态。
3. 如果右值不是常量，则清除目标变量原有常量状态。
4. 读取变量时，只有命中当前环境且该读取不是赋值左值、取地址目标、函数名、类名等特殊符号时才替换。

该实现不跨函数传播，不做分支合流，不做循环固定点。这样牺牲了一部分优化机会，但可以避免在别名、控制流和副作用不明确时改变语义。

---

## 5. 拷贝传播

拷贝传播处理“变量只是另一个变量的拷贝”的场景。当前实现只替换使用点，不删除中间变量声明、赋值或局部槽位。

典型形式：

```text
b = a;
return b;
```

优化环境记录 `b -> a`，后续读取 `b` 时替换为读取 `a`。

算法要点：

- 只记录普通变量到普通变量的拷贝关系。
- 链式拷贝会追踪到 canonical 源变量，例如 `c -> b -> a` 最终可替换为 `a`。
- 常量传播优先级高于拷贝传播；如果 `b` 已知是常量，则直接替换为常量。
- 写入源变量或目标变量都会使相关拷贝关系失效。

拷贝传播不处理数组元素、字段、对象属性、解引用结果或调用结果，因为这些表达式涉及别名和副作用风险。

## 6. 简单分支优化

简单分支优化基于常量条件裁剪控制流。

已支持的情况：

- `if` 条件恒为 true：保留 then 分支，删除 else 分支。
- `if` 条件恒为 false：保留 else 分支；没有 else 时替换为空块。
- 条件经常量折叠后为常量时参与裁剪。
- 无副作用条件下，then/else 分支结构等价时可合并。
- `while(false)` 删除循环体。
- `for(init; false; update)` 保留初始化，删除循环体和 update。

分支优化不会把被保留分支内部的传播状态合并回外层。原因是块级作用域、变量写入和控制流出口可能影响外部语义；首版只负责删除静态不可达分支，而不是做完整控制流数据流分析。

为了保持代码生成阶段的作用域行为，分支替换时尽量保留块级结构和符号表关联。

---

## 7. 公共子表达式消除

公共子表达式消除当前采用“语句级 CSE”。它只在单条语句内部查找重复出现的纯复合表达式，并插入编译器临时变量复用结果。

典型形式：

```text
return (a + b) * (a + b);
```

可改写为等价结构：

```text
__vbc_cse_0 = a + b;
return __vbc_cse_0 * __vbc_cse_0;
```

算法步骤：

1. 对单条语句完成常量折叠、传播、拷贝传播和分支处理。
2. 在优化后的语句中收集纯复合表达式。
3. 为表达式构造结构化签名，包含节点类型、运算符、变量身份、字面量值和类型。
4. 如果同一签名出现至少两次，则在语句前插入临时变量声明。
5. 将重复出现点替换为临时变量读取。

候选表达式必须满足：

- 表达式是复合运算，而不是单个变量或单个字面量。
- 表达式可证明纯。
- 表达式类型可明确推导。
- 同一语句中没有写入该表达式读取的变量。

当前跳过的表达式包括函数调用、对象构造、数组下标、字段访问、对象属性访问、解引用、显式 cast、`&&` 和 `||`。其中短路逻辑被跳过，是为了避免把原本按需求值的表达式提前求值。

CSE 临时变量会同步加入当前符号表，确保后续字节码生成可以分配局部槽位。

---

## 8. 简单内联

简单内联将极小函数调用替换为函数返回表达式。它是优化行为，不等同于语言级 `inline` 关键词。

可内联函数需要满足：

- 函数体足够简单。
- 函数体只有单个 `return expr`。
- 返回表达式可证明纯。
- 调用关系非递归。
- 参数替换不会引入副作用或求值顺序变化。

调用点也需要满足：

- 实参表达式无副作用。
- 实参可以安全替换形参。
- 返回值需要隐式转换时，内联结果保留等价转换边界。

内联候选使用表达式快照保存，而不是直接引用函数体 AST。这样可以避免函数体后续被其他优化改写后，污染已经登记的内联候选。

内联后的表达式会继续参与 AST 优化。因此，小函数内联常常会触发进一步的常量折叠、常量传播或 CSE。

---

## 9. 字节码窥孔优化

字节码优化层处理已经生成并解析跳转标签后的指令序列。它不再关心高级 AST 语义，只清理低层控制流冗余。

主要优化：

- 删除 `NOP`。
- 删除不可达指令。
- 删除跳转到下一条指令的无意义跳转。
- 将跳转链重定向到最终目标。
- 根据指令删除和重排结果更新跳转目标。
- 同步重映射行号表，保持错误定位合理。

字节码优化会多轮执行直到稳定。每轮优化都基于上一轮结果继续清理，因此删除无意义跳转后暴露出的不可达代码，也可以在后续轮次被删除。

---

## 10. 保守性边界

当前 `O1` 故意限制在局部、过程内和容易证明安全的范围内。

不属于当前 `O1` 的内容：

- 跨函数常量传播或拷贝传播。
- 跨基本块 CSE。
- 循环不变式外提。
- 循环固定点数据流分析。
- 复杂别名分析。
- 纯函数推断。
- 删除拷贝传播产生的中间变量和局部槽位。
- 对数组元素、结构体字段、对象属性和指针解引用的公共读取优化。

这些能力需要更完整的 IR、控制流图、别名模型或副作用模型支撑，更适合放入后续 `O2/O3` 优化层。
