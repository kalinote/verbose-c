## 项目协作规则

- 如果运行 PowerShell 脚本，或涉及 PowerShell 的中文输出，应先将 PowerShell 设置为 UTF-8 字符集。
- 所有 PowerShell 命令行操作都必须使用 PowerShell 语法，不要使用 Bash 语法。
- 代码修改应保持最小且高置信度。
- 不要创建只使用一次的函数。
- 新增简单函数应以 `"""内容"""` 格式写一行用法说明。
- 新增复杂函数应使用标准格式的 docstring。
- 修改代码前应检查是否会影响现有功能；确认没有影响后再修改。如果有影响但又必须修改，需要先向用户确认。
- 如果某个函数只有一两行代码逻辑，应直接将代码内联，不单独创建函数，除非后续明确会进行功能和逻辑扩展。
- 所有修改都应尽可能优先使用项目已安装的依赖。
- 如果必须安装新的依赖，在执行安装前需要先向用户确认，不要直接安装。
- 代码仅在关键逻辑上增加简短注释说明。
- 所有 Python 项目必须通过 `.venv` 虚拟环境运行，不要直接运行系统 Python。
- 除非特殊要求，不要擅自创建 Markdown 文档。
- 所有回答必须使用中文；如果有代码编辑，代码中的注释和日志输出也应使用中文。
